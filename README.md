# kabu-bot V3 — 学習型仮想売買bot + 専用ダッシュボード

日本株の仮想売買を GitHub Actions 上で完全自動運用し、成績を専用ページ
(GitHub Pages)で確認できるシステムです。Discord通知は廃止しました。

**これは仮想売買です。実際の発注は行いません。過去の成績は将来の利益を保証しません。**

---

## V3で何が変わったか(V1からの主な変更点)

| 項目 | V1 | V3 |
|---|---|---|
| 戦略 | 移動平均クロス1種類 | トレンド/逆張り/ブレイクアウトの3戦略 |
| 学習 | なし(固定ルール) | 戦略配分の自動学習+月次ウォークフォワード最適化 |
| 銘柄選定 | 固定5銘柄 | 週次で流動性・モメンタムから自動選定 |
| 確認方法 | Discord通知 | 専用ダッシュボード(レーダーチャート等) |
| リスク管理 | なし | 日次損失上限・DD上限での自動停止を実装済み(仮想売買では実害なしだが実弾移行に備える) |

## セットアップ手順(スマホブラウザで完結・約15分)

Secretsの登録は**不要**です。GitHub Pagesの有効化だけ行います。

### 1. リポジトリをPublicで用意する

このダッシュボードはGitHub Pagesという無料の公開ページ機能を使うため、
リポジトリを **Public(公開)** にする必要があります。仮想売買のデータなので
実害はありませんが、URLを知っていれば誰でも見られる状態になります。

- 新規にリポジトリを作る場合: 作成時に「Public」を選択
- 既存のPrivateリポジトリを使う場合: Settings を開き、一番下までスクロール
  （「Danger Zone」という赤い枠が出てくるまで）→
  「Change repository visibility」→ Public に変更

### 2. ファイル一式をアップロード

このzipの中身をリポジトリのルートにそのままアップロードしてください。
`.github/` フォルダ（ドットから始まる隠しフォルダ）の階層を崩さないよう
注意してください。

### 3. GitHub Pagesを有効化する

1. リポジトリの **Settings** タブを開く
   （タブが画面幅に収まらない場合は横にスクロール、またはブラウザの
   「デスクトップ用サイトを表示」に切り替えると見つけやすくなります）
2. 左側のメニュー（スマホでは上部のドロップダウンにまとまっていることが
   あります）から **Pages** を選ぶ
3. 「Build and deployment」という項目の **Source** を
   **「Deploy from a branch」** に設定
4. その下の **Branch** で **main** を選び、隣のフォルダ選択で **/docs** を選ぶ
5. **Save** をタップ
6. 1〜3分待つと、同じPagesの画面の上のほうに
   `https://あなたのユーザー名.github.io/リポジトリ名/`
   というURLが表示されます。これがあなた専用のダッシュボードです。
   スマホのホーム画面に追加しておくと次から一発で開けます

### 4. 初回実行

1. Actionsタブを開く（初回は「workflows enable」の確認が出ることがあるので
   緑のボタンで有効化）
2. 左側（またはメニュー内）の **weekly-screen** を開き、
   右側の **Run workflow** ボタン→さらに出てくる緑の **Run workflow** をタップ
3. 1〜2分待って緑の✓になったら、続けて **daily-trade** も同様に手動実行
4. ダッシュボードURLを開き、資産推移やレーダーチャートが表示されれば成功です
   （反映まで数分かかることがあります。表示されない場合はブラウザの
   再読み込みを試してください）

以降は平日15:50頃・土曜10:00頃に自動実行され、ダッシュボードも自動更新されます。

## ダッシュボードの見方

- **評価額 / 通算損益率**: 現在の仮想資産の状態
- **成績レーダー**: 勝率・損益率・DD耐性(下落への強さ)・取引効率・安定性の5軸
- **資産推移**: 開始からの評価額の推移
- **戦略配分**: トレンド/逆張り/ブレイクアウトへの資金配分比率
  （月次で自動更新。成績の良い戦略の比率が上がっていく）
- **保有銘柄・直近の取引**: 現在の持ち高と取引履歴
- **学習履歴**: 月次のパラメータ最適化で「更新」されたか「維持」されたか

## 運用の役割分担

**完全自動**: データ取得・売買判断・仮想約定・週次銘柄入替・月次学習・
ダッシュボード更新・異常時の自動停止（すべて0円・トークン消費ゼロ）

**あなたが定期的に行うこと**
- 週1回程度: ダッシュボードを開いて確認(1分程度)
- 月1回: 学習履歴を見て、最適化が「維持」続きなら戦略見直しをAIチャットで相談
- 年1回: `core/candidates.json` の銘柄リストを見直す
- 随時: 世界のAIトレード動向を取り入れたい時はAIチャットで相談し、
  `state/params.json` や `core/strategies.py` への変更として反映する

## ルール・パラメータの調整

`state/params.json` を編集すると各戦略のパラメータを直接変更できます
（ただし月次学習が自動的に上書きすることがあります）。
`config.json` では以下を調整できます。

| 項目 | 意味 |
|---|---|
| initial_capital | 仮想資金(円) |
| max_positions | 同時保有の上限銘柄数 |
| strategies_enabled | 有効にする戦略(trend/meanrev/breakout) |
| risk.daily_loss_limit_pct | 当日損失がこの%を超えたら新規買い停止 |
| risk.drawdown_limit_pct | 通算ドローダウンがこの%を超えたら全自動停止 |

## 緊急停止

`state/KILL_SWITCH` という名前の空ファイルを作成しコミットすると、
全ジョブが安全に何もせず終了するようになります。解除はファイルを削除するだけです。
ドローダウン上限を超えた場合はシステムが自動でこのファイルを作成します。

## 将来の実弾移行について

`core/broker.py` に立花証券e支店API接続用のクラス(`TachibanaBroker`)を
スタブとして用意しています。`config.json` の `"mode"` を `"paper"` から
`"demo"`(立花証券のデモ環境) や `"live"` に切り替える際は、このクラスへの
実装が必要です。移行前に必ず `daily_loss_limit_pct` / `drawdown_limit_pct` /
`max_position_pct` のリスク設定を見直してください。

## 既知の制約

- yfinanceのデータは非公式・遅延ありの参考値です
- 1株単位で仮想売買します（実際の日本株は原則100株単位）
- 手数料・税金・スリッページは考慮していません
- 深層強化学習は採用していません。データ量に対して過学習しやすいためで、
  代わりに戦略間配分のバンディット学習+ウォークフォワード検証という
  より堅実な構成を採用しています
- ダッシュボードは認証なしで公開されます(URLを知る人は誰でも閲覧可能)
